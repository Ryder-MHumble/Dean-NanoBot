# Student 输出质检门禁

在最终回复前，必须逐项检查并全部通过：

1. 结构完整：包含查询结论、学生样本段、必要时筛选项与能力边界说明。
2. 优先级完整：若展示多条学生记录，全部带 `P1/P2/P3`，按 `P1 -> P2 -> P3` 排序。
3. 链接口径正确：
   - 学生接口无外部 `sourceUrl`，不得伪造“原始链接”。
   - 若需追溯，只能给内部记录链接 `/api/v1/students/{student_id}` 并明确说明。
4. 隐私最小化：用户未明确要求时，不批量暴露 `email/phone`。
5. 不落地文件：默认直接回复，不创建本地 `.md`/`.markdown` 报告文件。
6. 长度受控：
   - Standard 模式（默认）：`P1<=12`、`P2<=6`、`P3<=2`。
   - Fast 模式（按需）：`P1<=5`、`P2<=3`、`P3<=1`。

若任一项不满足，先修正再发送。
