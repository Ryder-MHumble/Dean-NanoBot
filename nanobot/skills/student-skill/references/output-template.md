# 输出模板（学生信息摘要风格）

## 核心骨架（必须）

1. 查询结论（命中条数、是否调用筛选项/详情）
2. 优先级口径（`P1/P2/P3`）
3. 学生信息条目（多条时必须带 `P1/P2/P3`）
4. 能力边界说明（当存在部分支持/不支持/接口异常时必须出现）
5. 精化建议（2-4 条）

## 自适应段落（可选）

- 可选筛选项段（用户询问可选值时展开）
- 详情展开段（用户要求单个学生详情时展开）
- 冲突/歧义说明段（姓名重名、筛选冲突时展开）

## 模板

```markdown
### 学生信息检索结果
> 查询：{query}
> 命中学生 {student_count} 条；{detail_hint}
> 直接在当前会话返回结果，不保存本地 Markdown 文件

优先级口径：
- `P1`：直接命中学号/姓名或核心筛选条件
- `P2`：强相关补充样本
- `P3`：背景参考

学生信息方面：
- {item1_sentence}
- {item2_sentence}
- {item3_sentence}

可选筛选项方面，{options_sentence}。

能力边界说明：{boundary_hint}

可继续精化：
- `institution=高校名`
- `enrollment_year=2024`
- `mentor_name=导师名`
- `status=在读|毕业`

发送前执行 `references/quality-gate.md` 质检清单。
```

## 句子生成规则

- 单条学生句：
  - `P1|P2|P3` + `姓名（学校/年级/导师/状态）`
  - 若需要可追溯，可追加 `记录链接：/api/v1/students/{student_id}`
- 必须明确：学生接口不提供外部原始信源链接，记录链接仅用于内部追溯。
- 若用户未明确要求联系方式，不在主段落批量展示邮箱/电话。
- 不把内部记录链接说成“外部原始信源”。

## 模板稳定性（降成本/提稳定）

- 强约束字段固定：`P1/P2/P3`、链接口径、隐私最小化、边界说明。
- 段落组织允许自适应：可按问题重排筛选项/详情/冲突说明。
- 叙事允许灵活，不要机械填空；重点保持事实准确与字段口径一致。

## 能力边界说明模板

### 部分支持

```text
能力边界说明：当前可支持 {supported_parts}；暂不支持 {unsupported_parts}。已先返回可支持范围内结果。若需新增能力，请联系 AI 产品经理孙铭浩提需求。
```

### 暂不支持

```text
能力边界说明：当前学生信源与接口暂不支持 {unsupported_parts}。为避免误导，本次不编造结果。建议联系 AI 产品经理孙铭浩提需求。
```

### 接口暂时不可用

```text
能力边界说明：当前学生接口暂时不可用（{error_summary}）。为避免误导，本次不编造结果，也不自动切换到外部搜索。建议稍后重试；如需持续补齐能力，请联系 AI 产品经理孙铭浩。
```
