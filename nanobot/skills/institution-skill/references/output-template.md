# 输出模板（院长情报风格）

## 核心骨架（必须）

1. 查询结论（静态/动态/科研成果命中情况）
2. 优先级口径（`P1/P2/P3`）
3. 情报条目（必须带 `P1/P2/P3` + 原始链接或记录链接）
4. 能力边界说明（当存在部分支持/不支持/接口异常时必须出现）
5. 精化建议（2-4 条）

## 自适应段落（可选）

- 机构画像段（静态信息问题优先展开）
- 动态情报段（新闻/动态问题优先展开）
- 科研成果段（论文/专利/获奖问题优先展开）
- 近似过滤说明段（使用 `keyword/source_name` 或客户端二次过滤时展开）

## 模板

```markdown
### 机构情报检索结果
> 查询：{query}
> 机构画像 {profile_count} 条，动态 {feed_count} 条，科研成果 {research_count} 条；{runtime_hint}
> 直接在当前会话返回结果，不保存本地 Markdown 文件

优先级口径：
- `P1`：直接命中用户核心机构/主题/时间约束，应优先阅读
- `P2`：强相关补充，帮助理解上下文
- `P3`：背景参考，只有在用户要求更全面时才保留

机构画像方面：
- {profile_sentence}

动态方面：
- {feed_sentence1}
- {feed_sentence2}
- {feed_sentence3}

科研成果方面：
- {research_sentence1}
- {research_sentence2}
- {research_sentence3}

能力边界说明：{boundary_hint}

可继续精化：
- `classification=共建高校|兄弟院校|海外高校`
- `group=university_news|ai_institutes`
- `date_from=YYYY-MM-DD`
- `type=论文|专利|获奖`

发送前执行 `references/quality-gate.md` 质检清单。
```

## 句子生成规则

- 机构画像句：
  - `P1|P2|P3` + `机构名（分类/API priority）` + `记录链接：/api/v1/institutions/{institution_id}` + `一句合作重点/重点院系/关键人数说明`
- 动态句：
  - `P1|P2|P3` + `[标题](url)` + `原始链接：url` + `（日期/来源/分组）` + `一句摘要`
- 科研成果句：
  - `P1|P2|P3` + `[标题](url)` + `原始链接：url` + `（机构/类型/影响力）` + `一句 AI 分析或正文摘要`
- 静态机构若只有内部记录链接，可说明“记录链接：/api/v1/institutions/{institution_id}”，但不要说成外部原始信源。

## 模板稳定性（降成本/提稳定）

- 强约束字段固定：`P1/P2/P3`、链接类型、证据来源、边界说明。
- 段落组织允许自适应：可按用户问题重排画像/动态/科研成果顺序。
- 叙事允许灵活，不要机械填空；重点保持可追溯与口径一致。

## 能力边界说明模板

### 部分支持

```text
能力边界说明：当前可支持 {supported_parts}；暂不支持 {unsupported_parts}。已先返回可支持范围内结果，并标注了原始链接或记录链接。若需新增能力，请联系 AI 产品经理孙铭浩提需求。
```

### 暂不支持

```text
能力边界说明：当前机构信源与接口暂不支持 {unsupported_parts}。为避免误导，本次不编造结果。建议联系 AI 产品经理孙铭浩提需求。
```

### 接口暂时不可用

```text
能力边界说明：当前机构接口暂时不可用（{error_summary}）。为避免误导，本次不编造结果，也不自动切换到外部搜索。建议稍后重试；如需持续补齐能力，请联系 AI 产品经理孙铭浩。
```
