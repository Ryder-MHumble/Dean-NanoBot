# Sentiment Monitor

## 强制规则（MUST）

- 唯一允许的知识来源：当前 skill 文档、`references/`、远端 HTTP API 响应。
- 禁止读取当前 skill 目录之外的仓库代码、项目文档、导出文件、数据库或本地数据文件来补齐结果。
- 禁止用本地脚本作为默认取数路径；执行方式必须是“调用后端 API 取数”。
- 默认直接在当前会话输出最终报告；除非用户明确要求导出，否则禁止写本地 `.md`/临时报告文件。
- 默认主监控对象只有：`北京中关村学院`、`中关村人工智能研究院`。
- `深圳河套研究院`、`上海创智研究院` 只作为兄弟机构对比样本，默认不与我方案例混排。
- 每条正式入选案例都必须同时给出：
  - `P1/P2/P3`
  - `机构归属`
  - `高相关依据`
  - `原始链接`
  - `处理建议`
- 仅出现“中关村”或弱 hashtag，不构成高相关命中；没有 `content_url` 的条目不能进入正式案例。
- 输出前必须逐项通过 `references/quality-gate.md`。

## 核心目标

- 产出“风险处置优先”的高相关舆情推送，而不是数据看板。
- 只保留与北京中关村学院/中关村人工智能研究院直接相关的帖子。
- 对兄弟机构（深圳河套、上海创智）做风险处置对标，提供我方可借鉴动作。
- 给出读完就能执行的处置清单，而不是泛化建议。

## API 配置

- 服务基址：`http://10.1.132.21:8001`
- 舆情信息流：`GET /api/v1/sentiment/feed`
- 内容详情：`GET /api/v1/sentiment/content/{content_id}`
- 舆情概览：`GET /api/v1/sentiment/overview`
- 快捷日报：`GET /api/v1/reports/sentiment/latest`
- 自定义报告：`POST /api/v1/reports/generate`

详细字段见 `references/api-catalog.md`。

## 默认执行模式

### Standard（默认）

- 主体输出：我方高相关负面案例 + 兄弟机构对比 + 立即执行清单。
- 结果规模建议：
  - `P1 <= 3`
  - `P2 <= 3`
  - 兄弟机构对比案例 `<= 4`
  - 执行动作 `3-6` 条

### Fast（按需）

- 仅当用户明确要求“更快/更短”时启用。
- 结果规模建议：
  - `P1 <= 2`
  - `P2 <= 2`
  - 兄弟机构对比案例 `<= 2`
  - 执行动作 `3-4` 条

## 标准流程（SOP）

1. 先建高相关候选池
- 分别调用：
  - `GET /api/v1/sentiment/feed?keyword=北京中关村学院`
  - `GET /api/v1/sentiment/feed?keyword=中关村人工智能研究院`
  - `GET /api/v1/sentiment/feed?keyword=深圳河套研究院`
  - `GET /api/v1/sentiment/feed?keyword=上海创智研究院`
- 默认按 `publish_time desc` 拉最近内容；必要时补 `platform` 过滤。

2. 再做实体级高相关筛选
- 只保留直接提到目标机构全称、官方账号、或“别名 + 明确上下文”的内容。
- 以下情况必须剔除：
  - 仅出现“中关村”
  - 仅是话题标签弱命中
  - 只是在别的机构内容里顺带一提
  - 缺少原始链接
  - 证据不足，无法确认机构归属

3. 对正式候选补证
- 对所有 `P1/P2` 候选，调用 `GET /api/v1/sentiment/content/{content_id}`。
- 补正文上下文、评论证据、互动量，再决定是否正式入选。

4. 优先级排序
- 先看 `严重性`：投诉/举报/虚假/退款等优先。
- 再看 `传播性`：互动量、评论量、扩散性。
- 再看 `实体确定性`：官方账号/全称直提 > 别名 + 上下文 > 证据不足。

5. 兄弟机构对比
- 仅把深圳河套研究院、上海创智研究院作为对比样本。
- 默认只输出“代表性案例 + 对我方启发”，不替兄弟机构生成执行清单。

6. 降级策略
- `latest report` 只能做背景线索，不能当最终证据。
- 若 `latest=0` 但 `feed` 有高相关案例，必须继续生成处置报告。
- 若没有满足门槛的我方负面帖子，明确写：
  - `未发现满足高相关门槛的负面帖子`
- 禁止放宽到泛“中关村”内容补数。

## 输出规范

默认输出骨架：

1. `核心结论`
- 命中多少条高相关帖子，P1/P2 各多少条，是否剔除了低相关样本。

2. `我方高相关负面案例`
- 每条必须包含：
  - `P1/P2/P3`
  - `机构归属`
  - `平台`
  - `发布时间`
  - `帖子摘要`
  - `负面点摘要`
  - `高相关依据`
  - `原始链接`
  - `处理建议`

3. `兄弟机构对比`
- 每条必须包含：
  - `P1/P2/P3`
  - `机构归属`
  - `高相关依据`
  - `原始链接`
  - `对我方启发`

4. `立即执行清单`
- 必须是动作，不是态度。
- 每条建议都要能回指某个案例或某类已出现问题。
