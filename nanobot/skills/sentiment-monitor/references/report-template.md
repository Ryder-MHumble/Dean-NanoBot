# 舆情监测报告模板（双维度 v3.0）

本文档为双维度报告的结构说明和示例，供理解报告内容和验收使用。

---

## 报告结构

报告分为两个独立维度，互不交叉。

交付要求：
- 默认直接在当前会话或消息通道输出最终报告，不保存本地 `.md` 文件，除非用户明确要求导出。
- 报告中所有被突出展示的情报项，必须附优先级与来源链接信息。

优先级口径：
- `P1`：最值得立刻阅读或处理，直接命中当前监控问题或行动目标。
- `P2`：强相关补充，建议本周跟进。
- `P3`：背景参考，仅在需要更全面信息时保留。

核心骨架（必须）：
- 维度一和维度二（只要有数据就要出现）
- 重点条目必须带 `P1/P2/P3` 与原始链接
- 至少一个可执行行动清单段

自适应段落（可选）：
- 当风险较少时，可压缩风险段并扩展机会段
- 当竞品差距明显时，可扩展竞品对比与追赶策略
- 当数据不足时，可输出“数据不足说明 + 下一步采集建议”

## 维度一：官方账号运营分析

**数据来源**：source_keyword LIKE '@%' 的官方账号帖子及评论
**核心问题**：账号内容表现如何？用户在说什么？和竞品账号比怎么样？

### 1.1 账号概览

表格展示各账号的基本运营指标（帖子数/总互动/平均互动/正面占比）。

### 1.2 高互动帖子

每个账号 TOP 3 帖子，含互动分解（赞+评+转+藏）、优先级和原文链接。

### 1.3 评论主题洞察

用户评论中最高频的关键词及代表性评论（附情感标签）。

### 1.4 竞品账号对比

横向对比表 + 差距/优势洞察：
- ✅ 若我方领先：说明领先点
- ⚠️ 若竞品领先：说明差距量，建议研究竞品哪类内容互动高

### 1.5 内容运营建议

分两部分输出：

**本周选题建议（有数据支撑）**，最多 4 条，每条注明数据来源：

```
- P1 | 📌 **选题：#AI人才培养** — 全网已有 8 篇相关内容，平均互动 342，情感以 🟢 为主。
  建议本周在官方账号发布一条关于「AI人才培养」的内容，借势当前讨论热度。

- P2 | 📌 **借鉴竞品选题**：@河套研究院 的《从零开始的AI实践》获得 1.2K 互动 [🔗 原始链接]。
  可参考其角度，结合我方特色发布差异化内容。

- P1 | 📌 **延伸爆款内容**：《xxx》是近期互动最高的帖子（2.1K）[🔗 原始链接]。
  分析其话题切入角度，本周发布一条延续性内容（Part 2 / 相关话题）。

- P2 | 📌 **放大用户认可点**：评论中「专业」「推荐」出现频率高（用户说：「老师讲得很专业...」），
  说明用户对这类内容反应积极。建议专门围绕这些话题策划 1-2 篇内容。
```

**执行层面**，最多 3 条：

```
- 🎯 **回应用户疑虑**：评论中多次出现「贵」，发布一条专门解答这些疑虑的内容，或在评论区直接回复。
- 🎯 **提升 @xxx 互动率**：平均互动低于 100，下次发布时在结尾加一个问题引导评论。
- 🎯 **回复评论**：本周至少回复 5 条用户评论（优先选高赞评论）。
```

## 维度二：全网舆情洞察

**数据来源**：关键词搜索数据（中关村人工智能研究院、北京中关村学院等）
**核心问题**：全网在说什么？有哪些真实风险和机会？下一步做什么？

### 2.1 声量总览

简表（不超过 6 行）：帖子数/评论数/互动量/情感分布/平台分布。

### 2.2 关键风险

**原则**：只列高/中优先级风险，每条必须附：
1. 原文链接（可点击核实）
2. 相对当前问题的优先级（如 `P1` / `P2`）
3. 具体回应建议（不写"关注"，写"发布xx内容"或"私信xx"）

```markdown
#### P1 / 🔴 风险1：投诉、欺骗

> 《有没有被坑过的朋友？感觉被骗了...》

- 📱 平台：小红书 | 👤 作者：用户A | 📊 互动：345
- 原始链接：[查看原文](链接)

**回应建议**：
- 48 小时内：评估内容真实性，准备官方澄清；
- 若情况属实：直接联系作者私信道歉并说明改进计划；
- 若情况不实：发布简洁声明，附事实证据。
```

### 2.3 正向机会

可转发/借势的高互动正面内容，每条附优先级、原始链接、操作建议。

### 2.4 立即执行清单

3-5 条带优先级的 checkbox：

```markdown
- [ ] **【48h内 / P1】** 回应高风险内容《xxx》→ [原始链接](链接)
- [ ] **【本周内 / P2】** 转发高互动正面内容《xxx》（互动量 2.3K）→ [原始链接](链接)
- [ ] **【本周内】** 在声量最高的 B站 发布 1 条主动内容
```

## 写作原则

1. **具体 > 通用**：不写"加强内容运营"，写"下周在小红书发布 2 条话题为 #XX 的内容"
2. **数据支撑**：每条建议都能追溯到具体数据或内容
3. **可立即执行**：读完报告应该知道今天要做什么，明天要做什么
4. **链接必须有**：风险内容、机会内容和被重点点名的帖子都要附原文链接，方便核实
5. **优先级必须有**：重点内容至少要有 `P1/P2/P3` 或等价优先级标签
6. **少即是多**：不列不重要的内容，宁可报告短一点，也不要用篇幅凑数
7. **结构稳而不死**：保持核心骨架不变，但允许根据问题类型灵活重排段落

## 旧版章节参考（仅供历史对比）

### 1. Executive Summary（执行摘要）

**目的**: 为领导提供快速概览，重点突出需要关注的关键信息

**必须包含**:
- 总体舆情评级（正面/中性/负面）
- 关键数字指标（总提及量、平台数、互动量）
- 最重要的发现（2-3 条）
- 紧急行动项（如有高优先级风险）

**示例格式**:
```markdown
## Executive Summary

**总体舆情**: 🟢 正面 (68% 正面, 25% 中性, 7% 负面)

**关键发现**:
- 总提及量: 94 条（跨 4 个平台）
- 高互动内容: 3 条超过 1000 点赞
- 风险预警: 2 项需要关注

**紧急行动项**:
1. 处理小红书上关于招生透明度的疑虑
2. 监控微博上关于研究成果的讨论
```

### 2. Sentiment Overview（舆情概览）

**目的**: 展示整体情感分布和趋势

**必须包含**:
- 情感分布表格（正面/中性/负面的数量和百分比）
- 情感趋势可视化（文本图表）
- 与历史数据的对比（如可用）

**示例格式**:
```markdown
## Sentiment Overview

### 整体情感分布
| 情感 | 数量 | 占比 |
|------|------|------|
| 🟢 正面 | 64 | 68% |
| 🟡 中性 | 24 | 25% |
| 🔴 负面 | 6 | 7% |

### 情感趋势可视化
正面 | ██████████████ 68%
中性 | █████ 25%
负面 | █ 7%
```

### 3. Platform Analysis（平台分析）

**目的**: 提供各平台的详细分析，识别平台特定的模式

**每个平台必须包含**:
- 总内容数
- 整体情感倾向
- 平均互动量
- 热门内容（前3-5条）
- 热门话题/标签

**示例格式**:
```markdown
## Platform Analysis

### 小红书 (Xiaohongshu)
**总内容数**: 20
**整体情感**: 🟢 positive
**平均互动**: 245 次

**热门内容**:
1. "体验过三个AI研究院了" - 123 点赞, 46 评论
   - 作者: 用户A
   - [查看原文](链接)
   - 情感: 中性

**热门话题**: #研究院比较, #招生, #课程质量
```

### 4. Risk Alerts（风险预警）

**目的**: 识别和优先排序需要关注的风险项

**风险分级**:
- 🔴 **高优先级**: 需要立即行动的严重问题
  - 包含严重负面关键词（投诉、举报、欺骗等）
  - 高互动量的负面内容
  - 可能引发危机的内容

- 🟡 **中优先级**: 需要监控的潜在问题
  - 一般负面关键词
  - 中等互动量的负面内容
  - 需要澄清的误解

**每个风险项必须包含**:
- 内容标题/摘要
- 平台来源
- 情感评级
- 问题描述
- 互动量
- 作者信息
- 建议行动

**示例格式**:
```markdown
## Risk Alerts

### 🔴 高优先级

**1. "有没有正在读博的大佬呀？"** (小红书)
- **情感**: 🔴 negative
- **问题**: 检测到风险关键词: 担心, 质疑
- **互动量**: 125
- **作者**: 用户B
- **链接**: [查看详情](链接)
- **建议行动**: 发布详细的招生标准说明，举办公开答疑会
```

### 5. Trending Topics（热点话题）

**目的**: 识别当前热门讨论主题，把握舆论焦点

**必须包含**:
- 前5-10个热门话题
- 每个话题的：
  - 提及次数
  - 平均互动量
  - 主要情感倾向
  - 情感分布

**示例格式**:
```markdown
## Trending Topics

### 前5大热点话题

**1. #AI教育** (28 次提及)
- 平均互动: 342
- 主要情感: 🟢 positive
- 情感分布: positive 20, neutral 6, negative 2
- 关键讨论: 课程质量、教学方法、学习效果

**2. #研究院排名** (15 次提及)
- 平均互动: 256
- 主要情感: 🟡 neutral
- 情感分布: positive 7, neutral 6, negative 2
- 关键讨论: 与其他研究院对比、综合实力评估
```

### 6. Account Monitoring（账号监控）

**目的**: 识别高影响力账号和异常活动

**必须包含**:
- KOL 列表（前5-10个高影响力账号）
- 每个 KOL 的：
  - 用户名
  - 所属平台
  - 发布内容数
  - 总互动量
  - 平均互动量
- 账号健康度评估：
  - 垃圾账号检测
  - 机器人检测
  - 协同攻击检测

**示例格式**:
```markdown
## Account Monitoring

### 高影响力账号 (KOLs)

**1. 李永乐老师** (小红书)
- 发布内容数: 2
- 总互动量: 2500
- 平均互动: 1250/条
- 情感倾向: 正面

### 账号健康度
- ✅ 未检测到异常账号活动
- ✅ 未发现垃圾/机器人账号
- ✅ 未发现协同负面攻击
```

### 7. Recommendations（行动建议）

**目的**: 提供可操作的建议，支持决策

**必须包含三个时间层次**:

1. **即时行动（本周）**: 需要立即处理的事项
   - 回应高优先级风险
   - 处理紧急问题
   - 及时的公关响应

2. **短期策略（本月）**: 近期需要实施的策略
   - 内容运营优化
   - 社群互动增强
   - 平台布局调整

3. **长期规划（本季度）**: 持续性的战略举措
   - 品牌建设
   - 数据驱动优化
   - 系统性改进

**示例格式**:
```markdown
## Recommendations

### 即时行动（本周）
1. **处理高优先级风险**
   - 针对招生透明度疑虑，发布详细说明
   - 联系关键负面内容的发布者
   - 在官方渠道发布回应

### 短期策略（本月）
1. **内容运营**
   - 基于热门话题创作相关内容
   - 增加在小红书和抖音的发布频率
   - 与高影响力创作者建立合作

### 长期规划（本季度）
1. **品牌建设**
   - 明确差异化定位
   - 建立统一的品牌语言
   - 持续输出高质量内容
```

## 报告撰写最佳实践

### 1. 面向领导的沟通

- **简洁明了**: 领导时间有限，突出关键信息
- **数据驱动**: 用数字说话，避免空洞描述
- **可操作性**: 每个发现都应对应具体建议
- **优先级清晰**: 明确什么需要立即关注

### 2. 情感表达

- 使用表情符号增强可读性：
  - 🟢 正面
  - 🟡 中性
  - 🔴 负面
  - ✅ 正常
  - ⚠️ 警告
  - ❌ 错误

### 3. 数字呈现

- 大数字使用 K/M 简写: 1.2K, 3.5M
- 百分比保留1位小数: 68.1%
- 使用表格和文本图表增强可读性

### 4. 链接和引用

- 提供原文链接便于核实
- 引用关键内容时注明来源
- 保护用户隐私（必要时匿名化）

### 5. 语言风格

- 专业但易懂
- 避免过度技术化的术语
- 中英文混用时保持一致性
- 使用主动语态

## 常见问题

### Q: 如果某个平台没有数据怎么办？
A: 在平台分析部分简要说明"本期未发现相关内容"，不需要详细展开。

### Q: 如果没有风险项怎么办？
A: 明确说明"✅ 当前无风险预警"，这本身就是积极信息。

### Q: 报告应该多长？
A: 完整报告通常 1500-2500 字，确保包含所有关键信息但不冗长。

### Q: 如何处理敏感内容？
A: 在报告中适度概括，避免完整复制可能引起二次传播的负面内容。

### Q: 历史对比数据从哪来？
A: 首次报告不包含对比，后续报告可以对比前一天/周的数据。
