# Sentiment Report 质检门禁（高相关 v4.0）

在发送定向舆情报告前，必须逐项检查并全部通过：

1. 主段存在：必须包含 `核心结论`、`我方高相关负面案例`、`立即执行清单`。
2. 对比段存在：若命中了兄弟机构样本，必须包含 `兄弟机构对比`。
3. 高相关门禁：
   - 正式案例只能属于 `北京中关村学院` 或 `中关村人工智能研究院`。
   - 仅出现“中关村”或弱 hashtag 的内容，不得进入正式案例。
4. 相关性依据完整：每个正式案例都必须写 `高相关依据`。
5. 链接完整：每个正式案例和兄弟机构对比案例都必须有原始链接；没有 `content_url` 的内容只能剔除或转人工复核。
6. 动作可执行：清单必须是可落地动作，且能回指到具体案例或对比案例。
7. 不用空看板补位：
   - 若未命中高相关负面，必须明确写 `未发现满足高相关门槛的负面帖子`。
   - 禁止用泛“中关村”内容、统计表或机会项凑数。
8. 长度受控：
   - `Standard`：总长度建议 `<= 8000` 中文字符。
   - `Fast`：总长度建议 `<= 5500` 中文字符。
9. 不落地文件：默认直接回复或发送消息，不创建本地 `.md` 文件。

可使用 `scripts/validate_intel_report.py` 做快速机器校验：

```bash
python3 scripts/validate_intel_report.py \
  --max-chars 8000 \
  --require-primary-monitoring true \
  --require-relevance-reasons true \
  --file - <<< "$report_content"
```
