# 输出模板（院长早报风格）

## 核心骨架（必须）

1. 查询结论（命中与检索方式）
2. 优先级口径（`P1/P2/P3`）
3. 政策情报条目（必须带 `P1/P2/P3` + 原始链接）
4. 能力边界说明（当存在部分支持/不支持/接口异常时必须出现）
5. 精化建议（2-4 条）

## 自适应段落（可选）

- 政策机会段（当用户关注申报/资金/招商时展开）
- 方法说明段（当使用回退检索或近似过滤时展开）
- 对比段（当返回结果跨类别/跨来源时展开）

## 模板

```markdown
### 政策情报检索结果
> 查询：{query}
> 命中政策 {feed_count} 条，机会 {opp_count} 条；{fallback_hint}
> 直接在当前会话返回结果，不保存本地 Markdown 文件

优先级口径：
- `P1`：直接命中用户核心需求，且应优先阅读/处理
- `P2`：强相关补充，帮助理解背景或下一步
- `P3`：背景参考，只有在用户要求更全面时才保留

政策情报：
- {item1_sentence}
- {item2_sentence}
- {item3_sentence}

机会层面：
- {opp1_sentence}
- {opp2_sentence}

能力边界说明：{boundary_hint}

可继续精化：
- `category=国家政策|北京政策|领导讲话`
- `importance=紧急|重要|关注`
- `min_match_score=70`
- `source_name=机构名`

发送前执行 `references/quality-gate.md` 质检清单。
```

## 句子生成规则

- 单条政策句：
  - `P1|P2|P3` + `[标题](sourceUrl)` + `原始链接：sourceUrl` + `（日期/来源/分类/匹配度）` + `一句摘要或洞察`
- 单条机会句：
  - `P1|P2|P3` + `[名称](sourceUrl)` + `原始链接：sourceUrl` + `（机构/状态/匹配度）` + `资金/截止日`
- 禁止输出只有分析没有链接的条目；若确实缺失 `sourceUrl`，明确标注“原始链接缺失”。

## 模板稳定性（降成本/提稳定）

- 强约束字段固定：`P1/P2/P3`、原始链接、证据来源、边界说明。
- 段落组织允许自适应：可合并/拆分小节，但不得遗漏核心骨架字段。
- 叙事允许灵活，不要机械填空；重点保持事实可追溯和结构稳定。

## 能力边界说明模板

### 部分支持

```text
能力边界说明：当前可支持 {supported_parts}；暂不支持 {unsupported_parts}。已先返回可支持范围内结果，并标注了原始链接。若需新增能力，请联系 AI 产品经理孙铭浩提需求。
```

### 暂不支持

```text
能力边界说明：当前信源与接口暂不支持 {unsupported_parts}。为避免误导，本次不编造结果。建议联系 AI 产品经理孙铭浩提需求。
```

### 接口暂时不可用

```text
能力边界说明：当前政策接口暂时不可用（{error_summary}）。为避免误导，本次不编造结果，也不自动切换到外部搜索。建议稍后重试；如需持续补齐能力，请联系 AI 产品经理孙铭浩。
```
