# Policy 输出质检门禁

在最终回复前，必须逐项检查并全部通过：

1. 结构完整：包含查询结论、政策条目（可选机会条目）、必要时的能力边界说明。
2. 优先级完整：所有被展示的政策/机会条目都带 `P1/P2/P3`。
3. 链接完整：每条条目都给 `sourceUrl`；若缺失，明确标注“原始链接缺失”。
4. 事实可追溯：数字、日期、来源字段来自 API 响应，不自行补造。
5. 不落地文件：默认直接回复，不创建本地 `.md`/`.markdown` 报告文件。
6. 长度受控：
   - Standard 模式（默认）：`P1<=6`、`P2<=6`、`P3<=3`，总长度建议 `<= 6500` 中文字符。
   - Fast 模式（按需）：`P1<=3`、`P2<=3`、`P3<=1`，总长度建议 `<= 3000` 中文字符。

若任一项不满足，先修正再发送。
